# OpenClaw 游戏化学习系统 - 数据库设计

## 数据库架构

```
┌─────────────────────────────────────────┐
│           PostgreSQL                    │
├─────────────────────────────────────────┤
│  users表     - 学员信息                 │
│  activities表 - 学习活动                 │
│  badges表    - 成就定义                 │
│  user_badges表 - 学员成就               │
│  cards表     - 生成的卡片               │
└─────────────────────────────────────────┘
```

## 表结构

### 1. users 表 - 学员信息

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    github_id VARCHAR(255) UNIQUE NOT NULL,
    github_username VARCHAR(255) NOT NULL,
    email VARCHAR(255),
    avatar_url TEXT,
    points INTEGER DEFAULT 0,
    streak_days INTEGER DEFAULT 0,
    last_checkin DATE,
    level VARCHAR(50) DEFAULT '新手',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_users_github ON users(github_id);
CREATE INDEX idx_users_points ON users(points DESC);
```

### 2. activities 表 - 学习活动

```sql
CREATE TABLE activities (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    activity_type VARCHAR(50) NOT NULL,  -- commit, issue, pr, star, checkin
    points INTEGER NOT NULL,
    description TEXT,
    source_url TEXT,  -- GitHub URL
    metadata JSONB,  -- 额外信息
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_activities_user ON activities(user_id);
CREATE INDEX idx_activities_type ON activities(activity_type);
CREATE INDEX idx_activities_created ON activities(created_at);
```

### 3. badges 表 - 成就定义

```sql
CREATE TABLE badges (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) UNIQUE NOT NULL,
    icon VARCHAR(10) NOT NULL,
    description TEXT,
    condition_type VARCHAR(50),  -- points, streak, activities
    condition_value INTEGER NOT NULL,
    points_reward INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. user_badges 表 - 学员成就

```sql
CREATE TABLE user_badges (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    badge_id INTEGER REFERENCES badges(id),
    earned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    card_generated BOOLEAN DEFAULT FALSE,
    card_pushed BOOLEAN DEFAULT FALSE,
    
    UNIQUE(user_id, badge_id)
);

CREATE INDEX idx_user_badges_user ON user_badges(user_id);
```

### 5. cards 表 - 生成的卡片

```sql
CREATE TABLE cards (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    badge_id INTEGER REFERENCES badges(id),
    card_type VARCHAR(50),  -- achievement, progress, summary
    image_url TEXT,
    push_status VARCHAR(50),  -- pending, sent, failed
    push_channel VARCHAR(50),  -- telegram, discord, feishu, email
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    sent_at TIMESTAMP
);

CREATE INDEX idx_cards_user ON cards(user_id);
CREATE INDEX idx_cards_status ON cards(push_status);
```

### 6. learning_progress 表 - 学习进度

```sql
CREATE TABLE learning_progress (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    course_id VARCHAR(100) NOT NULL,
    lesson_id VARCHAR(100) NOT NULL,
    status VARCHAR(50) DEFAULT 'not_started',  -- not_started, in_progress, completed
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(user_id, course_id, lesson_id)
);

CREATE INDEX idx_progress_user ON learning_progress(user_id);
CREATE INDEX idx_progress_course ON learning_progress(course_id);
```

## 自动流程

### 1. GitHub Webhook 自动记录

```javascript
// webhook/github.js
app.post('/webhook/github', async (req, res) => {
  const event = req.headers['x-github-event'];
  const username = req.body.sender.login;
  
  // 记录活动
  await recordActivity(username, event, req.body);
  
  // 检查成就
  await checkAchievements(username);
  
  // 自动生成卡片
  await autoGenerateCards(username);
  
  // 自动推送
  await autoPushCards(username);
  
  res.json({ success: true });
});
```

### 2. 成就检查定时任务

```javascript
// cron/achievement-checker.js
// 每小时运行一次
cron.schedule('0 * * * *', async () => {
  const users = await getAllUsers();
  
  for (const user of users) {
    await checkAndAwardBadges(user.id);
    await generateAndPushCards(user.id);
  }
});
```

## 部署

### Docker Compose

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://user:pass@db:5432/gamification
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis

  db:
    image: postgres:15
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: gamification
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:
```
